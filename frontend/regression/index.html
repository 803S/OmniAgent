Phase B Regression

Browser Execution Fixtures

这组页面专门给 click / fill / select / press_key / fill_form / heal 做本地回归。 每张页都尽量把一个执行风险收敛成可重复、可观测、可录 workflow 的固定场景。

P0

Native Form

原生按钮、输入框、下拉框、按键提交，适合验证 native-first 主链。

 P0

Select-like Controls

combobox、listbox、datalist、date/time 输入，专门验证复杂控件 fallback。

 P1

Modal + Dynamic List

弹层、局部刷新、分页和陈旧索引风险，适合做失败诊断与自愈闭环验证。

建议回归顺序

  1. 先跑 python3 scripts/self_check.py。
  2. 先做 Native Form，确认基本 click/fill/select/press_key 没回退。
  3. 再做 Select-like Controls，观察 complex control 是否仍被识别成 select。
  4. 最后做 Modal + Dynamic List，确认失败时 trace 和 heal 信息是否足够解释。

使用提醒

  • 每个 fixture 都内置事件日志，便于核对动作是否真正触发。
  • 如果要录 workflow，优先从这里录，减少第三方页面噪音。
  • 若本轮改了执行链，至少重跑所有 P0 页面。
